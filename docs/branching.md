# Branching Rules

We do not commit directly to `main`.
All work is done on branches and merged through Pull Requests.

## Branch naming patterns
- feature/ui-*
- feature/storage-*
- feature/tasks-*
- docs/*

## Examples
- feature/ui-task-list
- feature/tasks-crud
- feature/storage-json-save
- docs/requirements-v1

## Workflow
1. Create a new branch from `main`
2. Make changes and commit to that branch
3. Open a Pull Request into `main`
4. Review your changes
5. Merge
